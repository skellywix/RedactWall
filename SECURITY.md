# Security Policy

RedactWall is a data-loss-prevention product sold to regulated financial
institutions. We treat vulnerability reports as a first-class input.

## Reporting a vulnerability

- Email **security@redactwall.example** (replace with the monitored security
  contact before publishing this repository) with a description, reproduction
  steps, and impact assessment. Use synthetic data only in reproductions —
  never real member, patient, or cardholder data.
- Do not open public GitHub issues for suspected vulnerabilities.
- We acknowledge reports within **2 business days** and aim to provide a fix
  or remediation plan within **30 days** for high-severity issues.

## Scope

In scope: the control plane (`server/`), detection engine
(`detection-engine/`), sensors (`sensors/`), AI gateway (`gateway/`),
packaging and install scripts (`scripts/`), and published artifacts.

Out of scope: denial-of-service against demo instances, findings requiring a
compromised admin workstation, and reports generated by automated scanners
without a demonstrated impact.

## Severity and disclosure

| Severity | Definition | Customer notification |
|----------|------------|----------------------|
| Sev1 | Shipped vulnerability exploitable in customer environments, or detection silently failing | Affected customers notified by email within 24 hours, with remediation guidance |
| Sev2 | Blocking defect in the current release with a workaround | Fix in the next patch release; noted in release notes |
| Sev3 | Other defects | Scheduled into the release train |

Security fixes are backported to the current and previous minor release
(see `docs/process/RELEASE_PROCESS.md`). Each security fix gets a `Security` entry in
`CHANGELOG.md` and a GitHub Security Advisory once customers have had a
reasonable upgrade window.

## Supply chain

- `npm audit --omit=dev` runs in CI on every push.
- Releases ship with a CycloneDX-style SBOM (`npm run security:package`).
- The audit log is hash-chained; integrity is verifiable with
  `node -e "console.log(require('./server/db').verifyAuditChain())"`.
